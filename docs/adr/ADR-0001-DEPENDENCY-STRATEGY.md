# ADR-0001: 商用依存関係戦略

- Status: Accepted
- Date: 2026-07-10

## Context

Forge3Dは将来App Storeで商用販売する。継続的なSDK利用料、突然のサービス停止、第三者固有形式へのロックインを避けたい。

## Decision

1. MVPはApple標準APIと自前コードで成立させる。
2. クラウドAPIは基本編集機能の必須経路にしない。
3. CADカーネルはGeometryKernel境界の後ろに置く。
4. OCCTはMVPへ組み込まず、将来の候補としてのみ扱う。
5. すべての外部依存を`DEPENDENCIES.md`で記録する。
6. プロジェクトには外部エンジン非依存のfallbackメッシュを保存する。

## Consequences

- 初期の精密CAD機能は限定される。
- 自前メッシュ／スカルプト資産が蓄積される。
- 将来のカーネル変更が製品全体の書き直しになりにくい。
- ライセンス審査を通過したカーネルだけを後付けできる。
